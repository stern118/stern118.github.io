◇DCにおけるアラートインシデント管理システム 2017年 6月 〜 2017年 12月
  1. システム概要
    • データセンター内でのインシデントを管理するシステム
    • 監視サーバから届いてくるメールを解析してインシデントとして登録し、
      そのインシデントを管理するのがメインの機能
      (RedmineやBackLogのようなものをイメージしていただければと思います)
  2. プロジェクト内容
    • Javaでのフルスクラッチ開発
    • 既存システムのリプレース
  3. 技術要素

    4. 自分は技術選定からは関われませんでしたが、以下の様でした。

    • 言語：Java
    • フレームワーク：Spring Boot
    • RDBMS：MySQL
    • ORM：MyBatis
    • APサーバ：WildFly
    • フロント：HTML/CSS, JavaScript
    • バージョン管理：SVN
  4. 自身の担当フェーズ
    • コーディング
    • 単体テスト実施
